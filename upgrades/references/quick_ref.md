# Upgrades - Quick Reference

## Triggers
- "upgrades"
- "what's new"
- "new features"

## 14 Steps
1. ANALYZE & RESEARCH
2. SEARCH WEB
3. RESEARCH ISSUES
4. CHECK CURRENT SETUP
5. CHECK HINDSIGHT
6. DETAIL EACH UPGRADE (pros, cons, cost, rating, priority, complexity)
7. COMPARE ALTERNATIVES
8. PRIORITIZE
9. VERSION TRACKING
10. AUTO-BACKUP
11. ROI/COST ESTIMATE
12. DEPENDENCY CHECK
13. ROLLBACK PLAN
14. SCHEDULED UPGRADES

## Output
- Current setup table
- Upgrade details with ratings
- Dependencies at risk
- Rollback plans
- Recommended schedule

## Key Points
- Always backup before upgrades
- Check dependencies first
- Document rollback steps
